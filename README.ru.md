# SymGContacts

**SymGContacts** — это полнофункциональный клиент двусторонней синхронизации Google Контактов для устройств на базе Symbian OS (S60 3rd Edition Feature Pack 1, 5th Edition, Symbian^3 / Anna / Belle).

Проект разработан в 2026 году и доказывает, что Symbian всё ещё может работать с современными облачными сервисами. Приложение использует актуальный **Google People API (+OAuth 2.0)** и напрямую взаимодействует с системной базой данных контактов Symbian (`cntmodel.lib`), обеспечивая надежную работу без использования устаревших или отключенных сервисов (таких как Mail for Exchange или SyncML).

## Ключевые возможности

*   **Двусторонняя "Умная" Синхронизация:** Приоритет всегда отдается последним изменениям (будь то правка на телефоне или в браузере).
*   **Синхронизация всех полей:** Поддерживаются имена, несколько номеров телефонов, email-адреса, почтовые адреса, названия компаний, должности, дни рождения и заметки.
*   **Пакетная выгрузка (Batch Upload):** Высокая скорость работы при добавлении новых контактов благодаря пакетным запросам к Google API.
*   **Дедупликация (Слияние):** Автоматический поиск и объединение локальных дубликатов контактов перед выгрузкой в облако.
*   **Фоновая работа:** Приложение может запускаться вместе с системой и тихо синхронизировать контакты по таймеру (каждые 15 минут), не мешая работе устройства.
*   **Современная безопасность:** Работает через OAuth 2.0 (Out-Of-Band flow) и поддерживает TLS 1.2.

## Системные требования

### Для работы приложения на устройстве:
*   Смартфон на базе **Symbian 9.2 (S60 3rd Edition Feature Pack 1)** или выше.
*   **ВАЖНО:** Для успешного подключения к серверам Google (`people.googleapis.com`) на устройстве **должен быть установлен патч, добавляющий поддержку TLS 1.2** на уровне Qt библиотек. Без этого патча приложение не сможет пройти стадию SSL Handshake и выдаст ошибку синхронизации. Скачать патч можно отсюда: https://nnproject.cc/qtls
*   Активное интернет-соединение (рекомендуется Wi-Fi для первой синхронизации большой базы).
*   *Обязательно:* Разблокированный смартфон (установлен RomPatcher+ с нужными патчами) для установки неподписанного `.sis` файла.

### Для сборки из исходников (Среда разработчика):
Сборка под Symbian требует настройки ретро-окружения. Рекомендуется использовать виртуальную машину с Windows XP, Windows 7 или Windows 10.

*   [Qt SDK 1.1.2](https://nnm.nnchan.ru/dl/sdk/Qt_SDK_Win_offline_v1_1_2_en.exe) (содержит Qt 4.7.3 для Symbian). 
*   **Nokia Symbian SDK** (идёт в комплекте с Qt SDK).
*   Компилятор **GCCE** или **RVCT** (входит в состав Qt SDK / Carbide.c++).

## Настройка проекта в Google Cloud

Так как приложение использует официальный API, вам потребуется создать собственный проект в Google Cloud Console для получения ключей доступа (Client ID и Client Secret).

1.  Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
2.  Создайте новый проект (например, "Symbian Sync").
3.  В разделе **"APIs & Services" -> "Library"** найдите и включите **Google People API**.
4.  Перейдите в раздел **"OAuth consent screen"**:
    *   Выберите тип **"External"**.
    *   Заполните обязательные поля (имя приложения, email).
    *   В разделе **"Test users"** обязательно добавьте свой Google-аккаунт (иначе вы не сможете войти).
5.  Перейдите в раздел **"Credentials"**:
    *   Нажмите "Create Credentials" -> "OAuth client ID".
    *   Выберите Application type: **"TVs and Limited Input devices"** (Это критически важно для работы метода авторизации OOB, который мы используем).
    *   Нажмите "Create".
6.  Скопируйте полученные **Client ID** и **Client Secret**. Вы введете их в приложении при первом запуске.

## Сборка проекта

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/Computershik73/SymGContacts.git
    cd SymGContacts
    ```
2.  Откройте проект `SymGContacts.pro` в **Qt Creator** (из состава Nokia Qt SDK).
3.  Убедитесь, что в настройках проекта (Projects) выбран профиль сборки для **Symbian Device** (Target: `Release`).
4.  Выполните очистку и генерацию Makefile:
    *   `Build -> Clean Project`
    *   `Build -> Run qmake`
5.  Соберите проект (`Build -> Build Project`) и запустите его. Не забудьте в свойствах запуска указать создание неподписанного (unsigned) sis-пакета.
6.  После успешной сборки Qt Creator сгенерирует `.sis` файл в папке сборки.

## Особенности архитектуры (Для разработчиков)

*   **Отказ от Qt Mobility:** В проекте принципиально не используется модуль `QContactManager` (Qt Mobility), так как он имеет проблемы с совместимостью на устройствах Samsung (Omnia HD) и Sony Ericsson (Satio/Vivaz). Вся работа с телефонной книгой написана на чистом **Symbian C++ (`cntmodel.lib`)** через прямые вызовы к `CContactDatabase`. TODO: В теории это даёт возможность в дальнейшем сделать backport под более старые версии Symbian, "отвязав" код от использования Qt библиотек (они здесь в основном для GUI).
*   **Изоляция исключений:** Все вызовы к капризному API Symbian обернуты в макросы `TRAPD` с аккуратным использованием `CleanupStack`, что полностью исключает паники (Panics) ядра ОС при чтении или записи битых контактов.
*   **Детерминированное хеширование:** Для определения того, изменился ли контакт (в телефоне или в облаке), приложение использует алгоритм SHA-1. Хэш строится из отсортированных полей контакта, что исключает ложные срабатывания (Upload) при изменении порядка номеров телефонов.
*   **Хранение состояния:** Данные о синхронизации (`ETags` и локальные хэши) безопасно хранятся в простом текстовом файле `sync_state_v2.txt` в папке `private` приложения.
